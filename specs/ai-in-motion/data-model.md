# Data Model for AI in Motion

## Robot State Data Model

### RobotState
- **Fields**:
  - `position`: Vector3 (x, y, z coordinates in meters)
  - `orientation`: Quaternion (w, x, y, z rotation)
  - `joint_states`: JointState[] (array of joint positions, velocities, efforts)
  - `sensor_data`: SensorData[] (array of sensor readings)
  - `timestamp`: float (ROS time when state was recorded)
  - `robot_id`: string (unique identifier for the robot)

- **Relationships**:
  - Belongs to: Robot
  - Associated with: NavigationGoal (current goal being executed)
  - Contains: JointState, SensorData

- **Validation Rules**:
  - Position coordinates must be within navigation bounds
  - Orientation quaternion must be normalized (unit length)
  - Joint states must match robot's joint configuration

- **State Transitions**:
  - `idle` → `moving` → `executing_task` → `idle`
  - `idle` → `error_recovery` → `idle`

### JointState
- **Fields**:
  - `name`: string (name of the joint)
  - `position`: float (position in radians for revolute joints, meters for prismatic)
  - `velocity`: float (velocity in rad/s or m/s)
  - `effort`: float (effort in Nm or N)

- **Relationships**:
  - Part of: RobotState
  - Belongs to: Robot (through RobotState)

### SensorData
- **Fields**:
  - `sensor_type`: string (lidar, camera, imu, etc.)
  - `timestamp`: float (ROS time of measurement)
  - `raw_data`: object (sensor-specific raw data)
  - `processed_data`: object (processed sensor information)
  - `frame_id`: string (coordinate frame of sensor)

- **Relationships**:
  - Part of: RobotState
  - Generated by: RobotSensor

## Voice Command Data Model

### VoiceCommand
- **Fields**:
  - `audio_data`: string (base64 encoded audio)
  - `transcription`: string (text transcription of audio)
  - `intent`: string (recognized intent like "move_to", "pick_up")
  - `parameters`: object (intent-specific parameters)
  - `timestamp`: float (time of command received)
  - `confidence`: float (confidence score of recognition)

- **Relationships**:
  - Processed by: CommandProcessor
  - Results in: RobotAction (if successful)

- **Validation Rules**:
  - Audio data must be in valid format
  - Intent must be recognized from predefined list
  - Parameters must match expected format for intent

- **State Transitions**:
  - `received` → `processing` → `processed` → `executed` / `rejected`

### CommandIntent
- **Fields**:
  - `name`: string (unique identifier for the intent)
  - `description`: string (human-readable description)
  - `parameters_schema`: object (expected parameters structure)
  - `associated_action`: string (corresponding robot action)

- **Relationships**:
  - Used by: VoiceCommand
  - Defines: RobotAction mapping

## Navigation Data Model

### NavigationGoal
- **Fields**:
  - `target_pose`: Pose (desired position and orientation)
  - `priority`: int (0-10, higher is more important)
  - `status`: string (pending, active, succeeded, failed, cancelled)
  - `created_time`: float (time goal was created)
  - `completed_time`: float (time goal was completed, if applicable)

- **Relationships**:
  - Associated with: Robot
  - Created by: CommandProcessor
  - Implemented by: NavigationSystem

- **Validation Rules**:
  - Target position must be within navigation map bounds
  - Orientation must be normalized quaternion

- **State Transitions**:
  - `pending` → `active` → `succeeded` / `failed` / `cancelled`

### Pose
- **Fields**:
  - `position`: Vector3 (x, y, z coordinates)
  - `orientation`: Quaternion (w, x, y, z rotation)

- **Relationships**:
  - Used in: NavigationGoal, RobotState

## AI Integration Data Model

### CognitivePlan
- **Fields**:
  - `natural_language`: string (original user command)
  - `plan_steps`: PlanStep[] (sequence of actions to execute)
  - `confidence`: float (overall confidence in plan)
  - `created_time`: float (time plan was generated)
  - `status`: string (planned, executing, completed, failed)

- **Relationships**:
  - Generated by: CognitivePlanner
  - Applied to: Robot
  - Based on: VoiceCommand

- **State Transitions**:
  - `planned` → `executing` → `completed` / `failed`

### PlanStep
- **Fields**:
  - `action_type`: string (navigation, manipulation, perception)
  - `parameters`: object (action-specific parameters)
  - `required_objects`: string[] (objects needed for this step)
  - `status`: string (pending, executing, completed, failed)

- **Relationships**:
  - Part of: CognitivePlan
  - Results in: RobotAction

## Assessment Data Model

### AssessmentResult
- **Fields**:
  - `student_id`: string (identifier for the student)
  - `module_id`: string (which module was assessed)
  - `score`: float (0.0 to 1.0 scale)
  - `feedback`: string (instructor feedback)
  - `completed_time`: float (time of assessment completion)
  - `artifacts`: string[] (paths to submitted code/data)

- **Relationships**:
  - Associated with: Student
  - Based on: ModuleAssessment

- **Validation Rules**:
  - Score must be between 0.0 and 1.0
  - Module ID must be valid curriculum module

## Docusaurus Homepage Data Model

### HeroSection
- **Fields**:
  - `title` (string): Main title text
  - `subtitle` (string): Subtitle text
  - `primaryButtonText` (string): Text for primary button
  - `secondaryButtonText` (string): Text for secondary button
  - `primaryButtonLink` (string): Link for primary button
  - `secondaryButtonLink` (string): Link for secondary button
- **Relationships**: Contains button and text elements
- **Validation**: All text fields required, valid links provided
- **State Transitions**: None (static component)

### FeatureCard
- **Fields**:
  - `icon` (string): Emoji icon representation
  - `title` (string): Feature title
  - `description` (string): Feature description
- **Relationships**: Belongs to FeaturesSection component
- **Validation**: All fields required, valid emoji icons
- **State Transitions**: None (static component)

### TranslationState
- **Fields**:
  - `currentLanguage` (string): Current display language ('en' or 'ur')
  - `originalContent` (object): Map of original English content by element identifier
- **Relationships**: Affects all page content elements
- **Validation**: Valid language codes, content preservation
- **State Transitions**: english → urdu → english (toggle)

### ContentElement
- **Fields**:
  - `elementId` (string): Unique identifier for DOM element
  - `englishContent` (string): Original English text
  - `urduContent` (string): Translated Urdu text
  - `elementType` (string): Type of element (h1, h2, p, span, etc.)
- **Relationships**: Part of TranslationState
- **Validation**: Element ID must be unique, content must exist in at least one language
- **State Transitions**: None (state managed by TranslationState)
